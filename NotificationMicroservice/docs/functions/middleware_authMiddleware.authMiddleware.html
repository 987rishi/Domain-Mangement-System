    Function authMiddleware

    • authMiddleware(req: Request, res: Response, next: NextFunction): void

      Middleware to handle user authentication based on a JSON Web Token (JWT).

      Parameters

      • req: Request

        The Express request object. It is expected to contain an Authorization header in the format "Bearer ".

      • res: Response

        The Express response object, used to send an error response if authentication fails.

      • next: NextFunction

        The next middleware function in the stack.

      Returns void

      Does not return a value but either sends an HTTP response or passes control to the next middleware.

      Remarks

      SECURITY WARNING: This middleware currently uses jwt.decode() which does not verify the token's signature. It only decodes the payload, making it vulnerable to tampering. But we are verifying the token in the api-gateway environment for protecting sensitive routes without signature verification. For a secure implementation, replace jwt.decode() with jwt.verify() and provide the appropriate secret or public key.

      If the token is decoded, its payload is attached to the req.user object, making it available to subsequent route handlers.

      Example

      // Protecting a user-specific route
      router.get('/my-profile', authMiddleware, (req, res) => {
        // req.user is now available here
        res.json({ user: req.user });
      });
